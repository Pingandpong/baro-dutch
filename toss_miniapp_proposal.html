Toss Mini App

한번에 정산 — 영수증 한 장, 몇 초 만에 모임 정산 끝

영수증을 찍으면 항목·가격을 자동 추출하고, 사람별 금액을 즉시 분배/공유합니다.

① 어떤 앱인지 (요약)

모임·여행·동호회에서 발생하는 비용을 OCR로 자동 인식하고, 항목을 사람별로 터치 분배하여 정산 결과를 링크로 공유하는 토스 미니앱입니다. 토스의 송금·요청 송금 흐름과 자연스럽게 연결되어, 채팅방에서 계산기 두드릴 필요가 없습니다.

  • 핵심 JTBD: “영수증을 찍기만 하면 사람별 금액을 몇 초 만에 나눠 공유하고 싶다.”
  • MVP 기능: 영수증 OCR · 사람 배정 UI(균등/비율/개별) · 정산 결과 생성/공유(영/한) · 수수료/할인/배달비 분배 토글 · (옵션) 1회 리마인더
  • 차별화: OCR → 배분 → 공유 3단 자동화로 토스 기본 송금 기능을 보완

② 기술 스택 (핵심 라이브러리)

클라이언트

  • React (Toss 미니앱 웹뷰) + TypeScript
  • UI: Gluestack UI (@/components/ui/*)
  • 상태/폼: Zustand · React Hook Form
  • 이미지: Web Canvas · EXIF 정규화

백엔드

  • Firebase (Auth · Functions · Storage · Firestore)
  • Serverless OCR 파이프라인 (Cloud Functions)
  • 로그/옵스: GCP Cloud Logging · Error Reporting

AI/OCR

  • 영수증 OCR: PaddleOCR or Google OCR (서버 사이드)
  • 항목 정규화 & 세금/봉사료 탐지: 규칙 + LLM 보정
  • 다국어(영/한) 요약/공유문 템플릿: LLM

토스 연계

  • 정산 결과에서 송금/요청 송금으로 딥링크 연결
  • 공유 링크로 재방문 시, 상태 복구 & 지연 분배 이어서 진행

※ 사용자의 기존 선호 스택(React Native/Expo/Gluestack/Firebase)을 유지하되, 미니앱 특성상 웹뷰 기반 React로 최초 구현 후 RN 래핑 가능.

③ 비즈니스 모델 (광고 / 구독 / IAP)

  • Basic (무료): 월 10건 OCR, 기본 분배(균등/비율), 링크 공유, 광고 노출
  • Premium (구독): 무제한 OCR, 항목 드래그 분배, 다국어(영/한), 분배 규칙 저장, 광고 제거
  • Team (IAP/좌석): 소모임·동호회 대상 그룹 기록/정산 히스토리, 멤버 초대, 역할(정산담당) — 월/연 단위
전환 포인트
OCR 월 10건 초과 시 업셀
수익 믹스
광고 30% · 구독 60% · IAP 10%
CAC 절감
토스 내 바이럴/공유 링크

④ 시장 가능성 (왜-지금)

  • 행태 전환: 송금은 이미 토스 내에서 이뤄짐 → 송금 이전 단계의 정산 자동화 수요가 증가
  • 경험 공백: 채팅방·스프레드시트로 수동 분배하는 불편이 여전히 큼
  • AI 보급: OCR/LLM 비용 하락으로 실시간 품목 분배가 현실화
  • 슈퍼앱 트렌드: 토스 미니앱 생태계에서 생활밀착 자동화는 채택률이 높음

초기 타깃: 대학생/사회초년생의 점심·회식, 여행/동호회. 재사용 빈도와 추천 가능성이 높습니다.

부록 — MVP 플로우 (요약)

  • 영수증 촬영 → OCR → 항목/가격 파싱 (+세금/봉사료 감지)
  • 모임원 추가 → 균등/비율/개별 모드 선택 → 항목 드래그로 배정
  • 정산 결과 생성 → 송금 메모 복사 → 공유 링크 생성(영/한)
  • (옵션) 미정산자 1회 리마인더 발송
보안/프라이버시: 이미지 일시 저장 후 텍스트만 보관, 개인식별정보 최소화, 만료 정책 적용.
